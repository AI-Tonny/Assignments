Спочатку я постворював папки app де зберігається вся логіка програми, models для моделей User(Користувач) News(Новина),
services для сервісів auth_service(вхід/реєстрація логіка), news_service(логіка для роботи з новиною), user_service(
логіка для роботи з користувачем), я би ще добавив admin_service(закинув би логіку для Адміна але в мене там було не
багато функцій (Чи так роблять? Створюють окремий сервіс для окремих ролей?)), потім створив папку schemas для news та
user, щоб використовувати для отримання даних з клієнта, використовував для валідації Pydantic(BaseModel, Field, EmailStr
також можна було використовувати кастомну валідацію для більш контролю даних (@field_validator("field_name")) але вирішив
не заморочуватися), створив папку utils для утиліт там в мене файл decorators.py там два декоратора token_required та
roles_required(цей декоратор використовується для того щоб вказати ролі, які можуть користуватися ендпоінтом (в реальних
проектах я думаю таке використовують, так?)), та папка routes в мене там всі роути які я потім приєдную до основного,
сама app містить __init__ де є функція яка збирає програму до купи та повертає обєкт API, як я дізнався це патерн який
називається фабрика додатків.
Нарахунок складнощів то це були відносні імпорти не зручно коли ти пишеш from ..(інша папка) import (...) в середині
файла іншої папки тому, що воно не підсвічує можливі папки після .. (я думаю це можна якось добавити), та коли ти
викликаєш напряму файл з таким імпортом буде помилка і говоритиму за імпорт ImportError: attempted relative import with
no known parent package ну я про це все почитаю.
Я зрозумів що краще створювати назву папку в snake_case або PascalCase щоб можна робити було абсолютний імпорт.
